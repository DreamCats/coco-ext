# prd-refined.md 产物模板

## 模板

```markdown
# 完善后的 PRD

## 基本信息
- **任务 ID：** {YYYYMMDD-简述}
- **原始需求：** {PRD 原文或一句话摘要}
- **创建时间：** {YYYY-MM-DD HH:MM}

## PRD 质量评分

### 初始评分
| 维度 | 分数 | 评价 |
|------|------|------|
| 功能明确度 | {N}/5 | {理由} |
| 边界完整度 | {N}/5 | {理由} |
| 交互/展示 | {N}/5 | {理由} |
| 验收标准 | {N}/5 | {理由} |
| 业务规则 | {N}/5 | {理由} |
| **总分** | **{N}/25** | |

### 完善后评分
| 维度 | 分数 | 评价 |
|------|------|------|
| 功能明确度 | {N}/5 | {理由} |
| 边界完整度 | {N}/5 | {理由} |
| 交互/展示 | {N}/5 | {理由} |
| 验收标准 | {N}/5 | {理由} |
| 业务规则 | {N}/5 | {理由} |
| **总分** | **{N}/25** | |

- **问答轮次：** {K} 轮

## 需求详述

### 功能描述
{合并问答后的完整功能描述}

### 边界条件
- {条件 1}
- {条件 2}
- ...

### 交互/展示
- **位置：** {在哪里展示}
- **样式：** {颜色、字号、布局}
- **交互：** {点击行为、动画效果}
（如有设计稿，标注链接）

### 业务规则
- {规则 1}
- {规则 2}
- ...

### 验收标准
- [ ] {验收项 1}
- [ ] {验收项 2}
- ...

## 问答记录

### 第 1 轮
| # | 问题 | 回答 |
|---|------|------|
| Q1 | {问题} | {回答} |
| Q2 | {问题} | {回答} |
| ... | ... | ... |

| # | 问题 | 回答 |
|---|------|------|
| ... | ... | ... |

## 术语映射

| PRD 术语 | 代码标识符 | 状态 |
|----------|-----------|------|
| {业务术语} | {代码标识符} | ✅ 已匹配 / ❓ glossary 中未找到 |

## 遗留问题
- [ ] {仍未确认的问题 1}
- [ ] {仍未确认的问题 2}
- 无 ← 如果全部解决
```

## 填写规则

1. **需求详述** 是核心产出 — 合并 PRD 原文 + 问答结果，形成完整可执行的需求描述
2. **问答记录** 保留原始 Q&A，供后续回溯
3. **术语映射** 方便 Phase A 直接使用，不用重新查 glossary
4. **遗留问题** 只记录确实没解决的，不要把"可能以后需要考虑的"也放进来
5. 如果打分后无需问答（≥ 20 分），问答记录部分写"无需问答，PRD 质量达标"
6. 如果是纯后端需求且交互/展示标 N/A，在评分表中注明
